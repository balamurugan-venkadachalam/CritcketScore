Authentication Object Explained
👤
Principal
Who is the user?
Represents the identity of the currently authenticated user. Usually holds the username (a String) or a full UserDetails object after authentication. Think of it as the answer to: "Who are you?"
Object principal = auth.getPrincipal();
// → "john_doe" or UserDetails object
🔑
Credentials
How did they prove identity?
The proof of identity — typically the password or token used to authenticate. After a successful login, Spring Security erases credentials from memory for security to prevent accidental exposure. Think of it as the answer to: "How did you prove it?"
Object creds = auth.getCredentials();
// → password (erased after auth)
🛡️
Authorities
What are they allowed to do?
A collection of roles and permissions granted to the user. Used by Spring Security to make authorization decisions — e.g. can this user access /admin? Common values: ROLE_USER, ROLE_ADMIN. Think of it as: "What can you do?"
auth.getAuthorities();
// → [ROLE_USER, ROLE_ADMIN]
ProviderManager is the most commonly used implementation of AuthenticationManager
